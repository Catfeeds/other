分析机器人

分析机器人在信息大爆炸时代，通过智能语义以及大数据分析技术，能在海量的资讯里及时挖掘您想要的信息。

本文档主要针对写作机器人SDK的服务进行试用说明

服务名称 简要描述
传播轨迹 分析文本在互联网上的传播路径并实时计算出传播轨迹
领域热词 于大数据分析，帮你及时获取实时领域热点关键词

分析机器人SDK文档

一. 开始使用

  • 1.1，引入js文件

    在页面中引入GRobot-JS-SDK.min.js文件，如下：

    <script type="text/javascript" src="http://robotsdk.giiso.com/SDK/js/GRobot-JS-SDK.min.js?se
cretId=C0001160003&secretKey=ca56e5ef-9b3c-43e5-8b42-e8672d2d8ee8">
  • 1.2，初始化GRobot实例

    在使用数据接口之前，必须调用GRobot.ready()方法，进行初始化操作

    GRobot.ready(function(){
    console.log('ready')
})

二. 接口调用方法

  • 2.1 ，调用接口的方法

    获取资讯数据方法为封装异步ajax的函数，使用异步回调函数得到结果，获取数据后逻辑请在回调函数中执行

    GRobot.getData({
    apiName: 'collector.search',

    data: {
        keyword: '王宝强'
    },

    success: function(data){
        console.log(data)
    },

    error: function(status){
        console.log(status)
    },

})
  • 传入字段的说明
    字段 是否必传 说明
    apiName 是 字符串，要调用的接口名称，务必填写正确（中间包含 符号 . 不可省略）
    data 否 接口请求参数，具体请看对应接口文档，如果接口无需参数，该字段可以不传
    success 是 成功回调，请求成功时回调函数，data为成功后接口返回数据，JSON对象
    error 否 失败回调，请求失败时回调函数，status为错误状态码

三. 接口文档说明

  • 1 ，传播轨迹

    接口名称：传播轨迹

    接口代码:analyzer.newsTransferLogByContent

    所需data参数：

    参数 必输 类型 备注
    content 是 string 新闻内容，不超过5000字
    size 否 string 最多显示传播轨迹数量，默认100

    返回数据JSON示例：

    {
    "code":"0",
    "msg":null,
    "data":[
        {
            "site":"搜狐网",
            "logo":"http://tj.giiso.com/logo/3.png",
            "time":"2017-11-20 00:00:00",
            "_id":"AV_VNVjBu20l5mywItLW",
            "title":"德国禁止儿童手表：儿童手表真的让我们的孩子安全吗？",
            "url":"http://www.sohu.com/a/205355922_179092"
        },
        {
            "site":"搜狐网",
            "logo":"http://tj.giiso.com/logo/3.png",
            "time":"2017-11-20 11:34:00",

    返回参数说明：

    参数名 类型 说明
    logo String 网站logo
    time String 发布时间
    title String 新闻标题
    site String 来源网站
    _id String 新闻id
  • 2 ，领域热词
